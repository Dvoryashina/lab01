| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Reporting a Vulnerability

Если вы обнаружили уязвимость безопасности в этом проекте, пожалуйста, сообщите об этом ответственно.

### Как сообщить об уязвимости

1. **НЕ создавайте публичный Issue** для уязвимостей безопасности
2. Отправьте описание уязвимости по email: 7gsc9zrnyv@privaterelay.appleid.com
3. Используйте тему письма: `[SECURITY] Lab01 Vulnerability Report`

### Информация для включения в отчёт

- Тип уязвимости
- Полное описание проблемы
- Шаги для воспроизведения
- Потенциальное влияние
- Предложения по исправлению (если есть)

### Процесс обработки

1. **Подтверждение получения** - в течение 48 часов
2. **Анализ уязвимости** - в течение 7 дней
3. **Разработка исправления** - в зависимости от серьёзности
4. **Публикация исправления** - после тестирования
5. **Публичное раскрытие** - после выпуска исправления

## Безопасность кода

### GPG подписи коммитов

Все коммиты в этом проекте должны быть подписаны GPG для обеспечения:
- Аутентичности автора
- Целостности изменений
- Невозможности подделки истории

Проверка подписи:
```bash
git log --show-signature
```

### Зависимости

Проект использует следующие зависимости:
- `typer` - официальная библиотека от tiangolo
- Все зависимости устанавливаются через официальный PyPI

Регулярно обновляйте зависимости:
```bash
pip install --upgrade typer
```

### Безопасная разработка

При разработке следуйте этим практикам:

1. **Не коммитьте секреты**
   - Токены
   - Пароли
   - API ключи
   - Приватные ключи

2. **Используйте виртуальное окружение**
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   ```

3. **Проверяйте входные данные**
   - Валидация пользовательского ввода
   - Санитизация строк
   - Обработка исключений

4. **Регулярно обновляйте зависимости**
   ```bash
   pip list --outdated
   ```

## Известные ограничения

### Текущие ограничения безопасности

1. **Учебный проект** - не предназначен для production использования
2. **Базовая валидация** - минимальная проверка входных данных
3. **Локальное выполнение** - предназначен для локального запуска

## Best Practices

### Для контрибьюторов

- Всегда подписывайте коммиты GPG
- Проверяйте код на уязвимости перед PR
- Следуйте принципу наименьших привилегий
- Документируйте все изменения, связанные с безопасностью

### Для пользователей

- Клонируйте репозиторий только из официального источника
- Проверяйте GPG подписи коммитов
- Используйте виртуальное окружение Python
- Обновляйте зависимости регулярно
